# Matka Photo Studio 予約管理システム 開発計画書

## 1. プロジェクト概要

### 目的
現在のAppSheet + GAS + n8nの複雑な多システム構成を、**単一のWebアプリケーション**に統合する。LINE連携、予約管理、顧客管理を1つのシステムで完結させ、運用の簡素化と信頼性向上を図る。

### 現状の課題
- AppSheet / GAS / n8n / Google Sheets / Google Calendar の5システムが連携しており、障害時の切り分けが困難
- LINE Webhook の受信先（n8n vs GAS）の競合問題
- スプレッドシートのLINE_UserID更新漏れによる通知未達
- n8nの実行コスト（ポーリング頻度の制限）
- AppSheetの操作ログ（誰が何をしたか）の追跡が困難

### ゴール
- **1つのWebアプリ**で予約〜顧客〜注文〜LINE通知を統合管理
- PC/スマホ両対応のレスポンシブUI
- 6人以上のスタッフが同時利用可能
- Google Sheetsを引き続きデータベースとして使用（既存データ資産を活用）

## 2. 技術スタック

| 項目 | 技術 | 理由 |
|------|------|------|
| フロントエンド | Next.js 14 + Tailwind CSS | SSR対応、レスポンシブ、高速 |
| バックエンド | Next.js API Routes | フロントと同一プロジェクト |
| データベース | Google Sheets API | 既存データ移行不要 |
| カレンダー | Google Calendar API | 既存カレンダー継続利用 |
| LINE連携 | LINE Messaging API | Webhook直接受信 |
| 認証 | NextAuth.js（Google OAuth） | 既存Googleアカウントでログイン |
| ホスティング | Vercel | 無料枠あり、自動デプロイ |
| PDF生成 | jsPDF or Puppeteer | 領収書・見積書 |
